What are Network Protocols?

With networking, a protocol is a standard way of doing certain actions and formatting data so that multiple devices can communicate and understand each other. These do not nearly justify how much protocols affects data translation and communication, but these are some of the most common protocols one might come across:

TCP/IP

(Transmission Control Protocol/ Internet Protocol)

These are a set of rules that allows different computers to communicate. The IP protocol ensures that a computer connected to the Internet has a specific “serial” number called an IP address. TCP decides how data should be broken into their individual packets.

HTTP

(Hypertext Transfer Protocol)

This protocol is used for sending web pages and other resources over the World Wide Web. It outlines how web browsers request and receive content.

HTTPS

(Hypertext Transfer Protocol Secure)

An extension of HTTP that adds a layer of security through encryption. A user will know that a website is secure if it contains the extra “s” after HTTP in the URL.

DNS

(Domain Name System)

This protocol deals with readable domain names (like www.example.com) and converts them into numerical IP addresses. This is essential for user-friendliness.

FTP

(File Transfer Protocol)

Used for transferring files between computers on a network. Used for uploading and downloading files to and from web servers.

SMTP

(Simple Mail Transfer Protocol)

This protocol is used for sending emails between servers. This is crucial for communication over email.
